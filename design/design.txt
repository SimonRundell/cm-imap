I would like develop a web-based IMAP and SMTP email client.

My preferred development stack is React+Vite+JavaScript+PHP running on Apache2 and MySQL.

I would like as much core email client functionality as possible:

* multiple users, and the possibility of multiple email accounts for each user. 
* Viewing of emails as both separate inboxes or a single unified inbox
* HTML and text preview of emails, to the side of the inbox list
* Replication of IMAP folders on the server
* Reply, Forward, Reply All etc  Reply would come from the inbox account which received it.
* Autoreply specifiable between dates
* Signatures with full HTML capability including graphics
* Email creation with embedding of graphics
* MIME attachments
* Rules for email management: movement to other folders, flags, priorty labels, autoreplies, autodeletion, send to spam folder based upon pattern matching in sender, recipient (text and email), email content pattern matching

Please determine the best design for this web app, and ask questions during the design phase, then author this app once you havethe satisfactory answers.

Authentication & Users
Login method: Username/password only, or also OAuth (Google, Microsoft)?
User management: Self-registration, or admin-created accounts only? Is there an admin panel needed? Admin Panel - Yes. Allow self-registration AND admin creation
Session handling: Standard PHP sessions, or JWT tokens? JWT please
IMAP/SMTP Credential Storage
Credential security: IMAP/SMTP passwords must be stored to allow background sync. Are you OK with AES-256 encryption at rest (server-side key), or do you need something stronger (e.g., user-derived key, meaning sync only works while logged in)? Server-side key please
Sync strategy: Should email sync happen in the background (via cron) so the inbox is pre-populated on login, or only on-demand when the user opens the app? Both please.
Email Storage
Local caching: Should email bodies/attachments be stored in MySQL + filesystem (faster, works offline-ish), or always fetched live from IMAP? MySQL + Filesystem please
Attachment storage: Filesystem (e.g., /var/www/attachments/) or MySQL BLOBs? As attachments but in a system configurable location
UI / Composition
Rich text editor: Any preference? TinyMCE, Quill, and CKEditor are the main options. TinyMCE is the most full-featured for email use. Go for TinyMCE
Threading: Should emails be grouped into conversation threads (Gmail-style), or shown as flat individual messages? Threaded please
Real-time updates: Browser notifications + polling for new mail, or is manual refresh acceptable? Browser notifications on new email but with manual refresh button as well.
Rules Engine
Condition complexity: Should rules support AND/OR logic between multiple conditions, or just a single condition per rule? AND/OR please
Regex support: Full regex pattern matching, or simpler "contains / starts with / ends with" matching for non-technical users? Simpler matching please
Deployment
Server environment: Is this a fresh Apache2 + PHP setup, or are there existing constraints (PHP version, php-imap extension availability)? Not sure.
Scale expectation: Roughly how many users and accounts? (e.g., personal/family use vs. team of 50?) Personal/Family use.
